Guía de Uso

Volver al Dashboard

1. Introducción

El HubSpot Fusionador detecta registros duplicados en tu portal de HubSpot (empresas o contactos) y permite fusionarlos de forma controlada. Antes de aplicar cambios reales, puedes ejecutar una simulación que muestra cómo quedaría el objeto resultante sin modificar HubSpot.

2. Primeros pasos

  1. Inicia sesión con tu usuario y contraseña.
  2. Crea un nuevo proyecto desde el dashboard.
  3. Configura el Access Token (PAT) de HubSpot con permisos de lectura y escritura sobre CRM.
  4. Selecciona el tipo de entidad: Empresas o Contactos.
  5. Pulsa Probar Conexión para verificar que el token funciona.

3. Criterios de fusión

En Criterios de Fusión (botón en la barra superior o panel del proyecto) puedes configurar:

Reglas de coincidencia

Cada regla indica qué propiedades deben coincidir todas a la vez. Por ejemplo:

  • name, email — fusiona solo si nombre y email son iguales (ideal para contactos "David" con emails distintos).
  • name, phone — fusiona si nombre y teléfono coinciden.
  • domain — fusiona empresas con el mismo dominio.

Puedes añadir varias reglas; cada una detecta duplicados de forma independiente.

Reglas de predominancia

Orden de prioridad para elegir el registro que sobrevive:

  • Propiedad informada — prioriza quien tenga rellena una propiedad (ej. código NAV).
  • Mayor nº de asociaciones — ej. num_associated_contacts o num_associated_deals.
  • Datos más recientes — ej. hs_lastmodifieddate.
  • Más propiedades rellenas — el registro con más información.

4. Simulación vs. Aplicación

Simular Fusión (recomendado)

Lee todos los registros de HubSpot, detecta grupos de duplicados y calcula el objeto fusionado resultante sin modificar nada. Puedes ver la vista previa de cada grupo con el botón Ver Simulación.

Aplicar Fusión (irreversible)

Ejecuta la fusión real en HubSpot mediante la API POST /crm/v3/objects/{type}/merge. Por seguridad, la fusión masiva sin filtros está bloqueada. Usa filtros (dominio, nombre, máximo de grupos) o confirma explícitamente una ejecución completa.

5. Filtros de ejecución

  • Dominio/email: limita la búsqueda a registros que contengan ese valor.
  • Nombre: filtra por coincidencia parcial en nombre o dominio (no distingue mayúsculas/minúsculas ni acentos).
  • Máx. grupos: limita el número de grupos procesados (útil para pruebas).

6. Roles de usuario

  • Usuario: ve y gestiona solo sus propios proyectos.
  • Administrador: ve todos los proyectos (con nombre del creador), gestiona usuarios y accede a logs globales.

7. Logs

Todas las acciones (login, creación de proyectos, simulaciones, fusiones, errores) quedan registradas en el panel de logs. Se actualizan automáticamente cada 10 segundos.

8. Token de HubSpot

Crea una Private App en HubSpot con los scopes: crm.objects.companies.read, crm.objects.companies.write, crm.objects.contacts.read, crm.objects.contacts.write. El token se almacena cifrado en la base de datos local.

9. Buenas prácticas

  1. Siempre ejecuta una simulación antes de aplicar fusiones.
  2. Empieza con filtros restrictivos y pocos grupos (maxGroups: 5).
  3. Revisa la vista previa del objeto fusionado en cada grupo.
  4. Consulta los logs si algo falla.
  5. Cambia la contraseña del admin por defecto en producción.
